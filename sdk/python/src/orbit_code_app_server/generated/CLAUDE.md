# sdk/python/src/codex_app_server/generated/

Auto-generated Pydantic models derived from the Codex app-server JSON schemas. Do not edit manually.

## Purpose

Contains the complete set of Pydantic models for all JSON-RPC request params, response types, notification payloads, and domain enums used by the Codex app-server v2 protocol. Also provides a notification registry that maps JSON-RPC method names to their corresponding Pydantic model classes.

## Key Files

| File | Role |
|------|------|
| `__init__.py` | Module docstring only |
| `v2_all.py` | All generated Pydantic models (~178KB); includes `ThreadStartParams`, `TurnCompletedNotification`, `ThreadItem`, `Turn`, `TurnStatus`, enums like `AskForApproval`, `SandboxMode`, `ReasoningEffort`, etc. |
| `notification_registry.py` | `NOTIFICATION_MODELS` dict mapping JSON-RPC notification method strings (e.g., `"turn/completed"`, `"item/agentMessage/delta"`) to their Pydantic model classes |

## Generated By

`sdk/python/scripts/update_sdk_artifacts.py generate-types` reads the JSON schema bundle from `codex-rs/app-server-protocol/schema/json/codex_app_server_protocol.v2.schemas.json` and produces these files.

## Imports From

- `pydantic` (BaseModel, Field, etc.)

## Exports To

- `codex_app_server.client` imports request/response models for RPC methods
- `codex_app_server.api` imports params models for the high-level surface
- `codex_app_server.models` imports notification models for the `NotificationPayload` union
- `codex_app_server.__init__` re-exports selected models as public API
