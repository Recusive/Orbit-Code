# codex-cli/

## Purpose

This is the npm package wrapper (`@openai/codex`) that serves as the user-facing entry point for installing and running the Codex CLI via npm/bun. It is a thin JavaScript launcher that resolves and spawns the correct platform-specific native Rust binary.

## What It Does

- Publishes to npm as `@openai/codex` with per-platform optional dependencies (e.g., `@openai/codex-linux-x64`, `@openai/codex-darwin-arm64`)
- The `bin/codex.js` entry point detects the host OS/architecture, locates the correct native binary from the vendor directory, and spawns it with signal forwarding
- Includes a Dockerfile for running Codex inside a sandboxed container with network firewall rules
- Contains build and release scripts for packaging native binaries into npm tarballs

## Key Files

| File | Role |
|------|------|
| `package.json` | npm package definition; declares `bin.codex` pointing to `bin/codex.js` |
| `bin/codex.js` | ESM launcher that resolves the platform binary and spawns it |
| `bin/rg` | DotSlash manifest for downloading ripgrep binaries per platform |
| `Dockerfile` | Container image definition for sandboxed Codex execution |
| `scripts/` | Build, packaging, and deployment scripts (see `codex-cli/scripts/CLAUDE.md`) |

## Imports / Dependencies

- **No runtime npm dependencies**: the package relies entirely on native binaries bundled in `vendor/`
- Optional dependencies: platform-specific `@openai/codex-*` packages that ship the native `codex` and `rg` binaries
- The native binary (`codex`) is built from the `codex-rs/` Rust workspace

## Exports

- The `codex` CLI command (via `bin/codex.js` in package.json `bin` field)
- npm tarballs for each supported platform, generated by `scripts/build_npm_package.py`

## Supported Platforms

| Target Triple | npm Package |
|---------------|-------------|
| `x86_64-unknown-linux-musl` | `@openai/codex-linux-x64` |
| `aarch64-unknown-linux-musl` | `@openai/codex-linux-arm64` |
| `x86_64-apple-darwin` | `@openai/codex-darwin-x64` |
| `aarch64-apple-darwin` | `@openai/codex-darwin-arm64` |
| `x86_64-pc-windows-msvc` | `@openai/codex-win32-x64` |
| `aarch64-pc-windows-msvc` | `@openai/codex-win32-arm64` |

## Relationship to Other Directories

- **`codex-rs/`**: provides the actual native Codex CLI binary that this package wraps
- **`scripts/stage_npm_packages.py`** (repo root): orchestrates downloading artifacts and invoking `codex-cli/scripts/build_npm_package.py`
