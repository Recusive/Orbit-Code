# codex-rs/tui/src/snapshots/

Insta test snapshots for top-level TUI module tests.

## What this folder does

Contains `.snap` files generated by the `insta` crate during test runs. These snapshots capture the expected rendered output of tests defined in the `App` module (`../app.rs`) and other top-level modules, including agent picker item names, clear UI states, model migration prompts, startup deprecation notices, and fast status rendering.

## What it plugs into

- **../app.rs**: Tests in the `App` module use `insta::assert_snapshot!()` to verify rendered output against these snapshot files.
- Snapshot files follow the naming pattern `codex_tui__app__tests__*.snap`.
