# codex-rs/tui/src/status/snapshots/

Insta test snapshots for status output tests.

## What this folder does

Contains `.snap` files generated by the `insta` crate during test runs. These snapshots capture the expected rendered output of the `/status` card under various test scenarios including cached rate limits, credits display, forked-from info, monthly limits, reasoning details, and more.

## What it plugs into

- **../tests.rs**: Tests in the status module use `insta::assert_snapshot!()` to verify rendered output against these snapshot files.
- Snapshot files follow the naming pattern `codex_tui__status__tests__*.snap`.
