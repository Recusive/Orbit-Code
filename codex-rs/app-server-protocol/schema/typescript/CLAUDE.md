# app-server-protocol/schema/typescript

## Purpose

Auto-generated TypeScript type definitions for the app-server protocol. Provides strongly-typed interfaces for TypeScript/JavaScript clients communicating with the app-server.

## Structure

- **Root `.ts` files** -- One file per shared/v1 protocol type (e.g., `ClientRequest.ts`, `ServerNotification.ts`, `InitializeParams.ts`, `AuthMode.ts`, etc.). Each file exports a single type or interface.
- **`index.ts`** -- Barrel re-export file that re-exports all types from this directory and subdirectories.
- **`serde_json/`** -- TypeScript definitions for serde_json helper types (e.g., `JsonValue.ts`).
- **`v2/`** -- TypeScript definitions for all v2 protocol types (hundreds of files).

All files begin with `// GENERATED CODE! DO NOT MODIFY BY HAND!`.

## Regeneration

Generated by `cargo run --bin write-schema-fixtures` or `just write-app-server-schema`. Do not edit manually.

## What It Plugs Into

- Imported by TypeScript clients (e.g., VS Code extension, web clients) to get typed protocol definitions.
- Validated by `tests/schema_fixtures.rs` to ensure they stay in sync with Rust source types.
- Generated by `src/export.rs::generate_ts()` using the `ts-rs` crate.

## Exports To

- TypeScript/JavaScript consumers that need typed interfaces for the app-server protocol.
