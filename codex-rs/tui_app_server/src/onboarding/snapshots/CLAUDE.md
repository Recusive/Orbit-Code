# codex-rs/tui_app_server/src/onboarding/snapshots/

Insta snapshot files for onboarding screen tests.

## What this folder does

Contains `.snap` files generated by the `insta` snapshot testing crate. Each file captures the expected rendered output of onboarding components (welcome screen, auth screen, trust directory prompt) under specific test conditions.

## What it plugs into

- **../**: Test functions in onboarding module files use `insta::assert_snapshot!()` to compare rendered output against these golden files.
- **insta** (dev-dependency): Manages snapshot creation, diffing, and review.

## Key files

All files follow the naming convention `codex_tui_app_server__onboarding__<submodule>__tests__<test_name>.snap`.
